# ロボットアーム開発ロードマップ

現在の状態:

- Fusion由来の固定表示URDFは、基本形状が合っている。
- 可動URDFのリンク割り当てと関節方向も、基本的に合っている。
- 次はエンドエフェクタ先端 `tool0` を定義し、FK/IKへ進む。

このロードマップでは、`tool0` 定義から実機で座標指定移動を行うまでの順番を整理する。

## Phase 1: tool0を決める

目的:

- 逆運動学で指定する「手先座標」がどこなのかを固定する。
- グリッパやエンドエフェクタが変わっても、ツール先端だけ差し替えられるようにする。

決めるもの:

```text
tool0 = 作業上の先端点
```

候補:

- グリッパの左右爪の中心
- ペン先
- 吸着パッド中心
- 何も付けていない場合は手首先端の中心

CAD/Fusionで測る値:

```text
最後の関節リンクからtool0までの x, y, z [mm]
```

成果物:

- `tool0` または `tool_link` をURDFへ追加
- `joints_template.csv` または生成スクリプトに、最後の固定jointを追加

完了条件:

- PyBulletで `tool0` の位置が意図した先端にある
- 全軸0度のとき、`tool0` がCAD上の先端位置と一致する

## Phase 2: 順運動学を確定する

目的:

- 関節角から `tool0` の位置姿勢を計算できるようにする。
- シミュレータ、URDF、実機の座標系を一致させる。

やること:

1. PyBulletから各関節角に対する `tool0` 座標を読む
2. 自作FKコード、またはURDFベースのFKコードを作る
3. 同じ関節角を入れて、PyBulletとFKコードの結果を比較する
4. 実機を同じ角度にして、実測位置と比較する

確認姿勢:

```text
all zero
j1 +20 deg
j2 +20 deg
j3 +20 deg
j6 +20 deg
j4 +20 deg
j5 +20 deg
```

成果物:

- `tools/fk_fusion_urdf.py` などのFK確認スクリプト
- FKの座標系定義メモ
- 実測との差分表

完了条件:

- PyBulletとFKスクリプトの `tool0` 座標が一致する
- 実機とシミュレータのずれが説明できる範囲に収まる

## Phase 3: 関節空間の安全な移動を作る

目的:

- 逆運動学の前に、指定した関節角へ安全に滑らかに移動できるようにする。

やること:

1. 現在角度を読む
2. 目標関節角との差分を計算する
3. 最大速度、最大ステップ角、関節リミットを守って補間する
4. 実機へ少しずつ送る

最初は単純な線形補間でよい。

```text
current_joint_angles -> target_joint_angles
```

後で必要なら、台形速度やS字加減速にする。

成果物:

- OpenCR側またはPC側の関節角移動コマンド
- 関節角リミット表
- 非常停止/トルクOFF手順

完了条件:

- `q_target = [0, 10, -10, 0, 5, 0] deg` のような関節角目標へ安全に動く
- リミット外の指令を拒否できる
- 急に大きく動かない

## Phase 4: 逆運動学の最初の実装

目的:

- 指定した `tool0` の位置へ移動するための関節角を求める。

最初の対象:

```text
position only IK
```

つまり、まずは手先姿勢は固定または無視して、`x, y, z` だけ合わせる。

候補:

- PyBulletの `calculateInverseKinematics`
- 数値IK
- 解析IKは後回し

最初に扱う入力:

```text
target_x_mm
target_y_mm
target_z_mm
```

出力:

```text
joint_1_deg ... joint_6_deg
```

選ぶべき解:

- 現在姿勢に近い解
- 関節リミット内の解
- 自己衝突しにくい解

成果物:

- `tools/ik_to_joint_angles.py`
- 目標座標から関節角を出す確認コマンド

完了条件:

- PyBullet上で、指定座標へ `tool0` が近づく
- 到達不能な座標を検出できる
- 実機へ送る前に関節リミットチェックが入っている

## Phase 5: IK結果を実機へ送る

目的:

- シミュレータで確認したIK結果を、実機のDYNAMIXELへ安全に送る。

手順:

1. 目標座標を入力
2. IKで関節角を計算
3. 関節リミットを確認
4. PyBulletで最終姿勢を表示
5. 問題なければ実機へ関節角目標を送る
6. 低速で移動
7. 実機の現在角度を読み返す

最初のテスト座標:

```text
現在姿勢のtool0から、X方向に +10 mm
現在姿勢のtool0から、Z方向に +10 mm
現在姿勢のtool0から、Y方向に +10 mm
```

大きな移動はまだしない。

成果物:

- PC側からOpenCRへ関節角を送るスクリプト
- IK結果を実機コマンドへ変換する処理
- 実機テストログ

完了条件:

- 小さな座標移動で、実機の手先が期待方向へ動く
- 逆方向に動く軸がない
- 危険な急動作がない

## Phase 6: 手先直線移動

目的:

- 始点から終点まで、手先が直線に近い軌道で移動するようにする。

やること:

1. 始点と終点の間を数mm刻みで補間する
2. 各点でIKを解く
3. 関節角列に変換する
4. 関節速度が大きすぎる点を検出する
5. 実機へ順番に送る

入力:

```text
start_xyz_mm
goal_xyz_mm
step_mm
```

成果物:

- `tools/cartesian_line_motion.py`
- 手先直線移動のシミュレーション確認

完了条件:

- PyBullet上で `tool0` が直線に近く移動する
- 実機で小さい直線移動が安全にできる

## Phase 7: 姿勢制御を追加する

目的:

- `tool0` の位置だけでなく、向きも指定できるようにする。

後回しでよい理由:

- 姿勢まで含めるとIKの難易度が上がる
- ロール軸や手首軸の符号・原点が少しでもずれていると原因切り分けが難しい

最初に追加する姿勢:

```text
手先を常に下向きに保つ
```

その後:

```text
任意のroll/pitch/yaw指定
```

成果物:

- position + orientation IK
- 姿勢誤差の表示

完了条件:

- 指定座標へ移動しながら、手先姿勢も大きく崩れない

## Phase 8: 使いやすい操作インターフェース

目的:

- 実験時に毎回スクリプトを書き換えなくても動かせるようにする。

候補:

- CLI
- 簡単なGUI
- ジョイスティック操作
- ティーチング/プレイバック

最初はCLIで十分。

例:

```sh
python tools/move_tool0.py --x 120 --y 0 --z 180
python tools/move_joint.py --j1 0 --j2 20 --j3 -30 --j4 0 --j5 10 --j6 0
```

成果物:

- `tools/move_tool0.py`
- `tools/move_joint.py`
- 実験用README

完了条件:

- 座標指定と関節角指定の両方がCLIからできる

## 推奨する実装順

最短で座標指定移動まで行く順番:

1. `tool0` をURDFに追加
2. PyBulletで `tool0` 座標を表示する
3. FK確認スクリプトを作る
4. 関節空間の安全な移動を作る
5. PyBullet IKで `x,y,z` だけ解く
6. IK結果をPyBulletで確認する
7. 小さい移動だけ実機へ送る
8. 直線移動へ拡張する
9. 姿勢制御を追加する

## 次にやる具体タスク

直近のタスクはこれ。

```text
tool0 の位置をCAD上で決める
```

決めたら、次の値を記録する。

```text
基準リンク:
最後の関節名:
tool0_x_mm:
tool0_y_mm:
tool0_z_mm:
tool0の意味:
```

例:

```text
基準リンク: wrist_pitch_link
最後の関節名: wrist_pitch_joint
tool0_x_mm: 80
tool0_y_mm: 0
tool0_z_mm: 0
tool0の意味: グリッパ爪先中心
```

この値が決まったら、URDF生成スクリプトに `tool0` 固定リンクを追加する。
