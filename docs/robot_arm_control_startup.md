# ロボットアームを動かし始める手順メモ

自作ロボットアームを最初に動かすときは、いきなり逆運動学で指定座標へ動かすよりも、関節単位の確認から順番に進める方が原因を切り分けやすい。

## 推奨する進め方

1. 関節ごとの角度指令
   - 各モータを低速・低トルクで少しずつ動かす。
   - 回転方向、ゼロ点、可動範囲、リミットを確認する。
   - 例: `joint_1 = 10 deg`, `joint_2 = -20 deg`

2. 順運動学の確認
   - 関節角から手先座標を計算する。
   - 計算した手先位置と、実機の手先位置が合うか確認する。
   - リンク長、座標系、関節の回転方向のズレをここで見つける。

3. 関節空間の軌道制御
   - 現在角度から目標角度まで、関節角を補間して動かす。
   - ステップ指令ではなく、速度・加速度を制限する。
   - 台形速度、S字加減速、五次多項式補間などが候補。

4. 逆運動学による位置指定
   - 指定した手先座標を関節角へ変換する。
   - 変換後の関節角へ、関節空間の軌道制御で滑らかに移動する。
   - 複数解がある場合は、現在姿勢に近い解を選ぶ。
   - 到達不能な座標、特異点、関節制限の処理が必要。

5. 手先の直線軌道
   - 始点と終点の間を直交座標系で細かく補間する。
   - 各補間点で逆運動学を解く。
   - 物を掴んだまま真っ直ぐ動かす場合や、作業軌道を指定したい場合に使う。

## 逆運動学以外の動かし方

- ジョイスティック・手動操作
  - 各関節、または手先速度をリアルタイムに指令する。

- ティーチング・プレイバック
  - 手動で姿勢や経路を記録し、後から再生する。

- ヤコビアンによる速度制御
  - 手先を「X方向へ一定速度で動かす」のように速度で指令する。

- 力制御・インピーダンス制御
  - 押し付け、接触、組み立て作業向け。
  - 最初の動作確認段階では優先度は低い。

- 軌道計画
  - 障害物や自己衝突を避ける経路を計画する。
  - MoveIt などのフレームワークで扱うことが多い。

- 学習ベース制御
  - カメラ入力や模倣学習から動作を決める。
  - 初期段階では切り分けが難しいため、後回しが無難。

## 最初に入れておきたい安全機能

- 非常停止できる状態を作る。
- ソフトウェア上の関節角リミットを入れる。
- 低速・低トルクから動かす。
- 電流制限や過負荷検知を使う。
- 起動時に現在角度とゼロ点を確認する。

## まずの目標

最初の実装目標は次の流れにする。

```text
関節角指令
  -> 順運動学の検証
  -> 関節空間の滑らかな移動
  -> 逆運動学で1点へ移動
```

この順番なら、逆運動学でうまく動かないときにも、原因が「幾何モデル」「座標系」「モータ制御」「軌道生成」のどこにあるか切り分けやすい。

## Step 1: 関節角指令の動作確認

OpenCR用スケッチ `sketches/opencr_dxl_check/opencr_dxl_check.ino` に、1軸ずつ角度で動かすためのコマンドを用意した。

角度の基準は、起動時の姿勢ではなく DYNAMIXEL の fixed raw zero とする。初期設定では各IDのゼロ raw 値をすべて `2048` にしている。

### コマンド

- `s`
  - DYNAMIXEL ID の疎通確認。

- `r`
  - 現在位置を raw tick で読む。

- `o`
  - 固定ゼロとして使っている raw tick を表示する。

- `a`
  - DYNAMIXEL の fixed raw zero からの関節角度を読む。

- `l`
  - 現在設定している関節角度リミットと、1回の最大移動角度を表示する。

- `j <id> <delta_deg>`
  - 指定IDの関節を、現在角度から相対的に動かす。
  - 例: `j 1 5` は ID 1 を現在角度から +5 deg 動かす。

- `m <id> <target_deg>`
  - 指定IDの関節を、fixed raw zero から見た目標角度へ動かす。
  - 例: `m 1 0` は ID 1 を DYNAMIXEL のゼロ角度へ戻す。

- `0`
  - 全モータを ID 1 から順に `0 deg` へ戻す。
  - 各軸は最大 `10 deg` ずつ段階的に動かす。
  - 現在角度がリミット外の場合、その軸は拒否して止める。

- `d`
  - トルクオフ。

### 最初の実機確認手順

1. ロボットアームが周囲にぶつからない姿勢にあることを確認する。
2. 非常停止、電源遮断、またはすぐに `d` を送れる状態にする。
3. シリアルモニタを `115200 baud` で開く。
4. `s` を実行して、想定IDがすべて見えていることを確認する。
5. `r` を実行して、現在位置が読めることを確認する。
6. `o` を実行して、固定ゼロ raw 値を確認する。
7. `a` を実行して、現在姿勢が fixed raw zero から何度ずれているか確認する。
8. `l` を実行して、リミットを確認する。
9. `j 1 2` のように、まず 1軸だけ 2 deg 程度動かす。
10. `a` で角度変化を確認する。
11. `m 1 0` で DYNAMIXEL のゼロ角度へ戻るか確認する。
12. 同じ確認を ID 2 以降でも行う。
13. 各軸の動作方向と安全範囲を確認した後で、`0` で全軸をゼロへ戻す。

初期設定では、各関節のリミットは fixed raw zero から `-45 deg` から `+45 deg`、1回の最大移動角度は `10 deg` にしている。実機の可動範囲が狭い軸は、スケッチ内の `JOINT_LIMIT_MIN_DEG` と `JOINT_LIMIT_MAX_DEG` を狭める。

DYNAMIXEL の raw `2048` をロボットの機械的な関節ゼロとして扱う。軸ごとに組付け上のズレを補正したい場合だけ、スケッチ内の `DXL_ZERO_RAW` に個別の raw 値を入れる。

### 実機で確認したモータ正方向

現在の `j <id> <delta_deg>` コマンドは、DYNAMIXEL の raw 値基準のモータ正方向で動く。順運動学や逆運動学で使う数学上の関節角とは、必要に応じて符号変換する。

| ID | 関節の意味 | `j <id> +2` で動く向き |
| --- | --- | --- |
| 1 | ベース旋回 | 上から見て反時計回り |
| 2 | 肩ピッチ | 肩が上がる |
| 3 | 肘ピッチ | 肘が下がる |
| 4 | 肘ロール | 後ろから見て肘が時計回りに回る |
| 5 | 手首ピッチ | 手首が下がる |
| 6 | 手首ロール | 後ろから見て手首が時計回りに回る |

その後、実機画像を確認して ID 6 は手先端のロールではなく、土台に近い上腕側のロール軸として扱う方針に変更した。

順運動学では、まず次の直感的な符号を採用する候補にする。

| ID | 数学上の正方向候補 | モータ符号 |
| --- | --- | --- |
| 1 | 上から見て反時計回り | `+1` |
| 2 | 肩が上がる | `+1` |
| 3 | 肘が上がる | `-1` |
| 4 | 後ろから見て反時計回り | `-1` |
| 5 | 手首が上がる | `-1` |
| 6 | 後ろから見て反時計回り | `-1` |

ただしロール軸の正方向は、座標系を決めた後に右手系と合わせて最終決定する。

## Step 2: 順運動学の確認

まずは6軸すべてを一気に扱わず、ベース旋回、肩ピッチ、肘ピッチの3軸で手首付近の位置を計算する。肘ピッチ軸と肘ロール軸が直線上にないため、ロール軸以降は後でオフセットを含めたモデルに分けて追加する。

### 測定済み寸法

単位は `mm`。

| 記号 | 意味 | 値 |
| --- | --- | --- |
| `L0` | ベース基準面から肩軸までの高さ | `106` |
| `L1` | 肩軸から肘軸までの長さ | `111` |
| `L2` | 肘ピッチ軸から手首基準点までの長さ | `70` |
| `L3` | 手首側からエンドエフェクタ先端までの長さ | `80` |

`L3` はエンドエフェクタによって変わる。

### 追加で測る寸法

次の寸法がないと、順運動学で手先位置を安定して計算できない。

| 記号 | 意味 |
| --- | --- |
| `E_pitch_to_roll_x` | 肘ピッチ軸から肘ロール軸までの水平方向オフセット |
| `E_pitch_to_roll_z` | 肘ピッチ軸から肘ロール軸までの高さ方向オフセット |

最初の3軸モデルでは、肘ロール軸のずれをいったん無視して `L2` だけで近似する。実機との差が大きい場合に、肘ピッチ軸から肘ロール軸へのオフセットを追加する。

### 最初に使う簡易3軸モデル

座標系は次で始める。

- 原点: ベース旋回軸の中心
- `X`: アーム正面方向
- `Y`: 左方向
- `Z`: 上方向

実機の全軸 `0 deg` 姿勢は、次の形として扱う。

- 肩から肘までのリンクは `+Z` 方向を向く。
- 肘から手首基準点までのリンクは `+X` 方向を向く。
- 手首以降の仮ツール長も `+X` 方向へ伸びる。

関節角は次の符号で扱う候補にする。

| 関節 | モータID | 符号 |
| --- | --- | --- |
| ベース旋回 | 1 | `+1` |
| 肩ピッチ | 2 | `+1` |
| 肘ピッチ | 3 | `-1` |

この段階では、手先ではなく「手首基準点」の位置を先に合わせる。エンドエフェクタ先端は `L3` が変わるため、後でツール長として足す。

### 簡易3軸順運動学スクリプト

`tools/fk_3axis.py` で、シリアルの `a` コマンドで読んだ ID 1, 2, 3 の角度から手首基準点を計算する。

```sh
python3 tools/fk_3axis.py --j1 0 --j2 0 --j3 0
python3 tools/fk_3axis.py --j1 0 --j2 10 --j3 0
python3 tools/fk_3axis.py --j1 0 --j2 10 --j3 -10
```

初期モデルでは、モータ角の符号を次のように変換している。

```text
base = +ID1
shoulder = +ID2
elbow = -ID3
```

### 可視化用URDF

現在の簡易6軸モデルを `urdf/robot_arm_simple_3axis.urdf` に出力した。

このURDFに含めているもの:

- ベース旋回
- 肩ピッチ
- 上腕ロール
- 肘ピッチ
- 肘ロール
- 手首ピッチ
- 手首基準点
- 仮ツール長 `L3=80 mm`

このURDFにまだ含めていないもの:

- 肘ピッチ軸と肘ロール軸のオフセット
- エンドエフェクタごとの正確な形状

肘ロールは、後ろから見て時計回りに回る ID 4 のモータ正方向を右手系候補に合わせるため、URDF上では `elbow_roll_joint = -ID4` としている。

手首ピッチは、ID 5 の `+` 方向が「手首が下がる」向きだったため、URDF上では `wrist_pitch_joint = +ID5` としている。

ID 6 は手先ロールではなく、画像で確認した土台に近い位置の上腕ロール軸として扱う。後ろから見て時計回りに回る ID 6 のモータ正方向を右手系候補に合わせるため、URDF上では `upper_arm_roll_joint = -ID6` としている。

ID4以降の軸間距離はまだ実測値ではなく、`L3=80 mm` を `30 mm + 50 mm` に分割した可視化用近似。
